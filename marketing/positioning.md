# Navide Positioning Brief

## Category

Navide is an **AI-native software engineering environment** and **the engineering instrument for the Agent era**.

It is not positioned as another code editor, terminal multiplexer, IDE plug-in, or chat panel. It organizes engineering work around goals, agents, sessions, context, exceptions, and evidence.

## Primary audience

Navide is built first for a solo engineer who owns an end-to-end software outcome and already uses coding agents heavily:

- Independent developers
- Technical founders
- Engineers who own an entire product area

Team and enterprise positioning is outside the first marketing prototype.

## Problem

AI increases the amount of execution one engineer can initiate, but the traditional IDE still assumes one human manually drives one primary stream of work. As agent count and session count grow, the new bottleneck becomes coordination, durable context, visibility, exception handling, and verification.

## Core promise

> **One engineer. An entire AI engineering force.**

Navide turns goals into coordinated, visible, and verifiable software work while the engineer retains responsibility for intent, architecture, risk, judgment, and final acceptance.

## Category contrast

> **The IDE organized code. Navide organizes engineering work.**

Traditional IDE:

```text
Engineer -> edits files and operates tools -> software
```

Navide:

```text
Engineer -> directs goals, agents, decisions, and evidence -> software
```

## Message hierarchy

### 1. Direct the work

Turn a goal into coordinated agents, independent sessions, roles, and configurable development stages.

### 2. Keep project intelligence

Recover workspace-scoped history, runs, handoffs, and context across sessions through the local `.agent-team/` layer.

### 3. Intervene with precision

Use Diff, editor, terminal, diagnostics, Git, tests, and review when ambiguity, risk, or engineering judgment makes direct control valuable.

## Three engineering loops

- **Genesis** — move from an idea to a first validated form.
- **Evolution** — continuously ship features, fixes, experiments, and quality improvements. This is the daily center of the product.
- **Intervention** — inspect and directly change outcomes when judgment or precision is required.

## Primary objections

### Is this only a terminal multiplexer?

No. Independent terminal panes are one execution surface. The product direction joins agent and session lifecycle, configurable pipelines, private project intelligence, coordination, visibility, exception handling, and engineering evidence into one system.

### Is Navide already a complete IDE replacement?

No. Navide is already used as the founder's primary environment and has a working local-first foundation. Complete professional replacement is the long-term direction, not a claim that every capability has shipped.

### Does local-first mean fully offline?

No. Navide's own orchestration and workspace state run locally. External agent CLIs, model providers, Git hosts, MCP servers, search, and update checks may communicate with third parties when used.

### Is `.agent-team/` team synchronization?

No. It is private, per-user workspace intelligence, excluded from Git. Shared source files and intentionally authored project documents remain the team-visible truth.

## Claim ledger

### Current and supportable

- Navide supports multiple independent coding-agent and terminal panes.
- Supported coding CLIs include Claude Code, Codex, Antigravity CLI, and Grok CLI.
- Supported sessions can be detected, persisted, rebuilt, and resumed.
- Pipelines can define stages, parallel slots, roles, prompts, questions, documentation queries, and completion sentinels.
- Workspace-scoped state, run events, handoffs, and compatible token summaries are stored under `.agent-team/`.
- Navide provides editor, Diff, terminal, diagnostics, Git, test, and review surfaces.
- Navide is local-first and does not require a Navide account.
- Navide currently supports macOS 13+ and is installed from source.
- The founder uses Navide as the primary environment for developing Navide. This is first-party dogfooding, not customer validation.

### Directional and always labeled

- Navide will replace the traditional IDE as the primary engineering environment.
- Navide will provide complete management by exception.
- Navide will detect conflicts, risk, idle sessions, and failures comprehensively.
- The Project Intelligence Layer will become inspectable, portable, redactable, and controllable.
- One engineer will be able to move a product from idea to trustworthy delivery without another IDE as the center of work.

### Not supportable without new evidence

- A signed or notarized public download is available.
- Navide provides a complete workspace sandbox.
- Navide is universally offline.
- Navide makes an engineer a specific multiple faster.
- Navide has customer adoption, retention, usage, or performance numbers.
- Named companies or individuals endorse Navide.
- Navide currently replaces every professional IDE workflow.

# Navide 宣言

[English](../manifesto.md) | 繁體中文 | [文件中心](README.md)

## 一種新的工程動力

蒸汽機帶來的影響，不只是讓單一勞動者做得更快。它將生產動力從人類肌肉中分離，進而創造新的機械、工廠、運輸系統、組織形式與生活方式。

人工智慧正在知識工作中造成類似的斷裂。對軟體工程而言，執行能力不再受限於一個人可以輸入、記住或依序完成多少工作。工程師可以指揮多個人工智慧 Agent，平行研究、規劃、實作、測試、審查並微調產品。

AI 模型是新的引擎。但只有引擎，還無法形成工業系統。力量必須透過可靠的機械、控制、記憶、回饋與運作方法，才能真正改變生產方式。

Navide 正是為了成為這套工程利器而打造。

> **一位工程師，一整支 AI 工程力量。**

## 超越傳統 IDE

傳統 IDE 建立在一個基本假設上：工程師親自完成大部分實作工作。因此，它的中心是檔案、Editor，以及手動輸入的 Command。

Agent 時代改變了工程工作的中心。

工程師愈來愈需要定義意圖、選擇方向、組織平行智慧、處理例外、判斷取捨並接受證據。編輯程式碼仍然重要，但它不再是人與軟體系統互動時唯一或永遠最主要的方式。

```text
傳統 IDE
工程師 → 編輯檔案、操作工具 → 軟體

Navide
工程師 → 指揮目標、Agent、決策與證據 → 軟體
```

Navide 的目標不是在舊模型上再加一個聊天框，而是建立接替它的新工作環境。

## 協調成為新的瓶頸

當執行能力稀缺時，工程師的主要限制是能夠依序理解、撰寫與測試多少程式碼。當許多 Agent 可以同時執行，另一組限制便成為主導問題：

- 誰負責每一項成果？
- 每個 Session 需要哪些脈絡與邊界？
- 哪些任務可以獨立進行，哪些必須等待？
- 哪些決策、變更與假設彼此衝突？
- 什麼時候應該將注意力交還給人類？
- 哪些證據能讓成果值得信任？

只有更多智慧，卻沒有協調，帶來的可能只是更多動作，而不是更好的工程。Agent 時代需要的不只是更強的模型，也需要一套能夠指揮並行工作，同時保留因果關係、品質與人類責任的運作環境。

## 個人就是一個工程組織

工業組織透過增加人員、部門、管理層、會議與流程來擴張。AI 讓另一種形式成為可能：以一個人的意圖為中心，動態組成工程組織。

Navide 使用者不只是接收建議的程式設計師，而是一支 AI 工程力量的領導者。Agent 可以承接產品分析、架構、實作、測試、審查與維護；工程師仍然對目的、限制、判斷與驗收負責。

目標不是移除人類，而是消除一個人的想法與實現它所需組織能力之間的歷史限制。

## 例外管理

工程師不應該批准每一個可逆的檔案修改或例行測試 Command，否則強大的系統仍會退化成緩慢的 Prompt 序列。但 Agent 也不應該成為不可見、無法控制的黑箱。

Navide 採用不同原則：

> **高度自主、全程可見、隨時可介入，並以證據負責。**

工程師設定目標、邊界、優先順序與不可逆決策。Agent 在這些邊界內持續工作。當需求歧義、風險、衝突、外部影響或人類判斷使介入變得有價值時，Navide 才將注意力交還給工程師。

## 私有專案智慧

專案不只是原始碼檔案。它還包括意圖、決策、未完成工作、Session 歷史、失敗嘗試、測試證據、Handoff 與累積理解。

Navide 將這份個人工程記憶保存在本機 `.agent-team/`。它屬於個別工程師，不屬於 Navide 雲端帳號，也不屬於共用 Git Repository。它讓新的 Session 與不同 Agent 能繼承脈絡，而不必將私人 Prompt、Token 資料或 Terminal 歷史變成團隊可見的原始碼 Artifact。

原始碼仍然是軟體專案的共用事實；`.agent-team/` 則是指揮專案之工程師的私有智慧。

## 先有證據，再擴大宣稱

革命性的方向不代表可以含糊宣稱。Navide 必須區分今天已經能運作的能力與仍然屬於目的地的方向。

第一項證據是 Navide 被用來開發 Navide。下一項證據必須來自其他工程師在自己的專案中指揮真實工作。關於取代、自治、速度、安全或採用程度的宣稱，只能隨著產品行為與獨立證據逐步擴大。

標準很簡單：展示工作流程、揭露能力邊界，並讓成果附帶證據。

## 我們的方向

Navide 從軟體工程開始，因為整個生產循環都能以數位形式表示並驗證：需求變成任務、Agent 修改程式碼、測試產生證據、Git 記錄成果，而實際軟體可以直接執行。

任務很明確：

> **打造 Agent 時代的工程利器。**

當一位工程師能夠從一個想法出發，指揮 AI 工程力量進行建立與持續演進，在需要時精準介入，並在不回到傳統 IDE 作為主要環境的情況下交付可信軟體，Navide 才算真正成功。
